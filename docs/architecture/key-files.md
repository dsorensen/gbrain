# Key files ŌĆö full reference

> The complete per-file contract reference for gbrain: every module with its
> exports, flags, and invariants, in current-contract (condensed) form. This is
> the doc the terse `## Key files` map in `CLAUDE.md` points to. It is NOT loaded
> into agent context ŌĆö consult it (or the code) when you touch a specific file.
> For verbatim per-version history (PR numbers, decision tags, migration
> lineage), see `docs/architecture/key-files-history.md`.

> Each entry below states the file's **current** contract (exports, flags, invariants). Per-version history ŌĆö PR numbers, decision tags, migration lineage ŌĆö is archived verbatim in `docs/architecture/key-files-history.md` and is deliberately not loaded into agent context.

- `src/core/operations.ts` ŌĆö Contract-first operation definitions (the foundation). Exports upload validators `validateUploadPath`, `validatePageSlug`, `validateFilename`, plus `matchesSlugAllowList(slug, prefixes)` (glob: `<prefix>/*` matches recursive children; bare `<prefix>` matches exact only). Every `Operation` carries `scope?: 'read' | 'write' | 'admin'` + `localOnly?: boolean`; `sync_brain`, `file_upload`, `file_list`, `file_url` are `admin + localOnly` (rejected over HTTP). `OperationContext.remote` is a REQUIRED field (untrusted unless strictly `false`); `OperationContext.allowedSlugPrefixes` is the trusted-workspace allow-list set by the dream cycle; `OperationContext.auth?: AuthInfo` is threaded through HTTP dispatch for scope enforcement. `put_page` enforces: when `viaSubagent` and `allowedSlugPrefixes` set, slug must match the allow-list; else the legacy `wiki/agents/<id>/...` namespace check applies. Auto-link skipped only when `remote=true && !trustedWorkspace`. Trust-boundary call sites use fail-closed semantics (`ctx.remote === false` for trusted-only sites, `ctx.remote !== false` for untrust-unless-explicit-false). Exports `sourceScopeOpts(ctx)` ŌĆö precedence ladder for source-scoped reads (federated array `ctx.auth.allowedSources` > scalar `ctx.sourceId`/`ctx.auth.sourceId` > nothing); every read-side op handler routes through it so a source-bound OAuth client can't see neighboring-source rows.
- `src/core/engine.ts` ŌĆö Pluggable engine interface (BrainEngine). `clampSearchLimit(limit, default, cap)` takes an explicit cap. Exports `LinkBatchInput` / `TimelineBatchInput` for the bulk-insert API (`addLinksBatch` / `addTimelineEntriesBatch`). `BrainEngine` has a `readonly kind: 'postgres' | 'pglite'` discriminator. Methods include `batchLoadEmotionalInputs(slugs?)`, `setEmotionalWeightBatch(rows)` (composite-keyed on `(slug, source_id)`), `getRecentSalience(opts)`, `findAnomalies(opts)`, `listAllPageRefs(): Promise<Array<{slug, source_id}>>` (cross-source enumeration, ordered by `(source_id, slug)`), `getAdjacencyBoosts(pageIds): Promise<Map<number, AdjacencyRow>>` (graph-signals stage), `refreshPageBody(slug, sourceId, compiled_truth, timeline, content_hash)`, `migrateFactsToCanonical(phantomSlug, canonicalSlug, sourceId)`. `PageFilters` has `sort?: 'updated_desc' | 'updated_asc' | 'created_desc' | 'slug'` + `PAGE_SORT_SQL` whitelist. `SearchOpts` + `PageFilters` add `sourceIds?: string[]` (federated read axis; `WHERE source_id = ANY(...)` when set, scalar `sourceId` fast path otherwise). `traverseGraph(slug, depth, opts?)` and `traversePaths(slug, opts?)` accept `opts.sourceId` / `opts.sourceIds`; `traverseGraph` opts also takes `frontierCap?: number` (per-iteration recursive-CTE cap). Exports `TraverseGraphOpts`. `SearchResult` has optional attribution fields (`base_score`, `backlink_boost`, `salience_boost`, `recency_boost`, `exact_match_boost`, `graph_adjacency_boost`, `graph_cross_source_boost`, `session_demote_factor`, `reranker_delta`). Two REQUIRED methods: `deletePages(slugs, {sourceId}): Promise<string[]>` (returns slugs actually deleted) and `resolveSlugsByPaths(paths, {sourceId}): Promise<Map<path,slug>>` ŌĆö `sourceId` REQUIRED on both; both short-circuit on empty input and throw when input exceeds `DELETE_BATCH_SIZE`.
- `src/core/engine-constants.ts` ŌĆö single source of truth for engine batch-sizing constants. Exports `DELETE_BATCH_SIZE = 500`, consumed by both engines' `deletePages` + `resolveSlugsByPaths` and by the sync delete + rename loops. Lives outside `engine.ts` to avoid circular imports.
- `src/core/search/graph-signals.ts` ŌĆö per-query graph-signals helper. `applyGraphSignals(results, engine, opts)` runs as the 4th post-fusion stage (after backlink/salience/recency). Three boosts: `ADJACENCY_BOOST=1.05` (page is linked from 2+ OTHER top-K results), `CROSS_SOURCE_BOOST=1.10` (linked from 2+ DIFFERENT sources; dormant in single-source brains), `SESSION_DEMOTE=0.95` (3+ results from same chat session; keep the highest-scoring one at full score, demote the rest). All three inherit the floor-ratio gate preventing weak pages from getting boosted past strong ones via popularity. `computeScoreDistribution(results)` emits min/p25/p50/p75/p95/max + `reorder_band_width`. `sessionPrefix(slug)` extracts the chat-session anchor. Pure `pairedBootstrapPValue(deltas, resamples, rng)` exported for eval gates. Test seam via `adjacencyFn` DI. Fail-open: any error logs via `logGraphSignalsFailure` (JSONL audit) and returns the input array unchanged.
- `src/core/search/hybrid.ts` extension ŌĆö `runPostFusionStages` extended with a 4th stage (`graphSignalsEnabled`, `onGraphMeta`, `onScoreDistribution`). `base_score` stamped at function entry idempotently (captured ONCE before any boost stage mutates `score`). Each post-fusion stage stamps its multiplier: `applyBacklinkBoost` ŌåÆ `backlink_boost`, `applySalienceBoost` ŌåÆ `salience_boost`, `applyRecencyBoost` ŌåÆ `recency_boost`. `applyReranker` stamps `reranker_delta` (positive = improved). `applyExactMatchBoost` (in `src/core/search/intent-weights.ts`) stamps `exact_match_boost` when fired. Per-stage attribution powers `gbrain search --explain`; `formatResultsExplain` reads each field without coupling to stage ordering.
- `src/core/search/explain-formatter.ts` ŌĆö renders `SearchResult[]` as a multi-line per-result breakdown for `gbrain search --explain`. Reads every boost-stamping field. Handles the "no boosts applied" empty path. 4-decimal precision with trailing-zero strip.
- `src/core/search/mode.ts` extension ŌĆö new `graph_signals: boolean` knob in `ModeBundle` (defaults: `conservative=false`, `balanced=true`, `tokenmax=true`). `KNOBS_HASH_VERSION` folds a `gs=` parts entry so a graph-on cache write can't be served to a graph-off lookup. `SearchKeyOverrides` + `SearchPerCallOpts` + `loadOverridesFromConfig` + `SEARCH_MODE_CONFIG_KEYS` + `resolveSearchMode` + `attributeKnob` all carry the new field. Opt-out per-brain: `gbrain config set search.graph_signals false`.
- `src/core/audit/audit-writer.ts` ŌĆö shared JSONL audit primitive. Exports `createAuditWriter({kind, recordSchema})` returning `{log, readRecent}` plus shared helpers `computeIsoWeekFilename(kind, now?)` and `resolveAuditDir()` (honors `GBRAIN_AUDIT_DIR`). ISO-week file rotation; best-effort writes (stderr warn on failure, never throws); read-path scans current-week + previous-week files. Backs `rerank-audit.ts`, `audit-slug-fallback.ts`, `shell-audit.ts`, `supervisor-audit.ts`, `phantom-audit.ts`, and the `graph-signals-failures` audit (`logGraphSignalsFailure`).
- `src/core/cli-options.ts` extension ŌĆö `CliOptions` gains `explain: boolean`. `parseGlobalFlags` recognizes `--explain` anywhere in argv (stripped before command dispatch). `src/cli.ts` `formatResult` for `search` + `query` routes to `formatResultsExplain` when set; falls through to JSON / human formatters otherwise.
- `src/commands/search.ts:gbrain search stats` extension ŌĆö `graph_signals` section (enabled/source/failures_count/failures_by_reason). JSON envelope adds a `graph_signals` sibling property; `_meta.metric_glossary` adds `graph_signals.enabled` + `graph_signals.failures_by_reason`. Reads `search.graph_signals` config first, falls back to the mode default.
- `src/commands/doctor.ts` extension ŌĆö `graph_signals_coverage` check wired into both `runDoctor` (local) and `doctorReportRemote` (HTTP/JSON thin-client path). Reads `search.graph_signals` config first, falls back to mode default; silent `ok` when disabled. Computes inbound link coverage on the page set; warns at <10% with `gbrain extract all` fix hint; `ok` at Ōēź30% ("fire on most queries") and 10-29% ("fire occasionally"), each with the percentage embedded in the message.
- `src/core/engine-factory.ts` ŌĆö Engine factory with dynamic imports (`'pglite'` | `'postgres'`).
- `src/core/pglite-engine.ts` ŌĆö PGLite (embedded Postgres via WASM) implementation, all 40 BrainEngine methods. PGLite-specific DDL (pgvector, pg_trgm, triggers). `addLinksBatch` / `addTimelineEntriesBatch` use multi-row `unnest()` with manual `$N` placeholders. `connect()` wraps `PGlite.create()` in a try/catch emitting an actionable error and releasing the lock on failure so the next process can retry. `searchKeyword` / `searchKeywordChunks` multiply `ts_rank` by the source-factor CASE at chunk grain; `searchVector` is a two-stage CTE (inner `ORDER BY cc.embedding <=> vec` keeps HNSW usable, outer re-ranks by `raw_score * source_factor`; inner LIMIT scales with offset). `initSchema()` calls `applyForwardReferenceBootstrap()` BEFORE replaying SCHEMA_SQL ŌĆö probes for forward-referenced columns/tables the embedded blob needs and adds only what's missing (no-op on fresh/modern brains; threads the DDL connection so probes run inside the advisory-lock scope). `getBrainScore` returns 100/100 with breakdown 35/25/15/15/10 when `pageCount === 0`. `disconnect()` uses snapshot+early-null (null instance fields before any await) in a try/finally guaranteeing lock-release, keeping close-then-release order. Exports `classifyPgliteInitError(message): 'bunfs' | 'macos-26-3' | 'unknown'` + `buildPgliteInitErrorMessage(verdict, original)` routing the init-error hint by failure shape. Implements `deletePages`/`resolveSlugsByPaths` via `slug = ANY($1::text[])` array-param binding; throws when input exceeds `DELETE_BATCH_SIZE`, returns `RETURNING slug` for `deletePages`.
- `src/core/postgres-engine.ts` ŌĆö Postgres + pgvector implementation (Supabase / self-hosted). `addLinksBatch` / `addTimelineEntriesBatch` use `INSERT ... SELECT FROM unnest($1::text[], ...) JOIN pages ON CONFLICT DO NOTHING RETURNING 1` ŌĆö 4-5 array params regardless of batch size, sidesteps the 65535-parameter cap. `searchKeyword` / `searchVector` scope `statement_timeout` via `sql.begin` + `SET LOCAL`. `getEmbeddingsByChunkIds` uses `tryParseEmbedding` so one corrupt row skips+warns. `searchKeyword` / `searchKeywordChunks` / `searchVector` apply source-aware ranking by inlining the source-factor CASE and hard-exclude clause from `src/core/search/sql-ranking.ts`; `searchVector` is a two-stage CTE carrying `p.source_id` innerŌåÆouter. `_savedConfig` retains the connect config; `reconnect()` tears down + recreates the pool (called by supervisor watchdog after 3 consecutive health-check failures). `executeRaw` is a single-statement passthrough (no per-call retry). `connect()` applies `resolveSessionTimeouts()` as connection-time startup parameters. `countStaleChunks()` + `listStaleChunks()` server-side-filter on `embedding IS NULL`; `upsertChunks()` resets both `embedding` AND `embedded_at` to NULL when chunk_text changes without a new embedding. `initSchema()` calls `applyForwardReferenceBootstrap()` BEFORE replaying SCHEMA_SQL on the same probe set as PGLite, running the probe path on the DDL connection threaded from `initSchema`. `disconnect()` is idempotent (the `_connectionStyle` field tracks owned-pool vs shared-singleton; second call on an instance-pool engine is a no-op). `getBrainScore` returns 100/100 with breakdown 35/25/15/15/10 when `pageCount === 0`. `deletePages(slugs, {sourceId}): Promise<string[]>` via `DELETE FROM pages WHERE slug = ANY($1::text[]) AND source_id = $2 RETURNING slug`; `resolveSlugsByPaths` via `SELECT slug, source_path FROM pages WHERE source_path = ANY($1::text[]) AND source_id = $2`. FK cascades through `content_chunks`/`links`/`tags`/`raw_data`/`timeline_entries`/`page_versions`; `files.page_id` + `links.origin_page_id` go SET NULL. Both throw when input exceeds `DELETE_BATCH_SIZE` and short-circuit on empty input.
- `src/core/cjk.ts` ŌĆö Single source of truth for CJK detection. Exports `CJK_RANGES_REGEX`, `CJK_SLUG_CHARS` (character-class fragment), `CJK_SENTENCE_DELIMITERS` (`ŃĆé’╝ü’╝¤`), `CJK_CLAUSE_DELIMITERS` (`’╝ø’╝Ü’╝īŃĆü`), `CJK_DENSITY_THRESHOLD = 0.30`, `hasCJK(s)`, `countCJKAwareWords(s)` (30% density threshold), `escapeLikePattern(s)` (escapes `%`, `_`, `\\` for `ILIKE ... ESCAPE '\\'`). BMP-only ranges (Han / Hiragana / Katakana / Hangul Syllables). Consumers: `expansion.ts`, `sync.ts:slugifySegment`, `operations.ts:validatePageSlug + validateFilename`, `chunkers/recursive.ts`, `pglite-engine.ts:searchKeyword + searchKeywordChunks`.
- `src/core/audit-slug-fallback.ts` ŌĆö Weekly ISO-week-rotated audit JSONL at `~/.gbrain/audit/slug-fallback-YYYY-Www.jsonl`. `logSlugFallback(slug, sourcePath)` fires when `importFromFile` falls back to a frontmatter slug because `slugifyPath` returned empty. `readRecentSlugFallbacks(days)` reads the last N days for `gbrain doctor`'s `slug_fallback_audit` check. Honors `GBRAIN_AUDIT_DIR` via `resolveAuditDir()`.
- `src/core/embedding-pricing.ts` ŌĆö `EMBEDDING_PRICING` map keyed `provider:model` for the post-upgrade reindex cost estimate. Sibling to `anthropic-pricing.ts`. Entries: OpenAI text-embedding-3-large ($0.13/1M), 3-small ($0.02/1M), ada-002 ($0.10/1M), Voyage 3-large ($0.18/1M), 3 ($0.06/1M). `lookupEmbeddingPrice(modelString)` returns a tagged union (`known` with price / `unknown` with provider name); `estimateCostFromChars(charCount, pricePerMTok)` uses 3.5 chars/token approximation. Unknown providers degrade to "estimate unavailable".
- `src/core/post-upgrade-reembed.ts` ŌĆö Pure functions backing the `gbrain upgrade` chunker-bump cost prompt. `computeReembedEstimate(engine, model)` queries SQL `COUNT(*)` + `COALESCE(SUM(LENGTH(compiled_truth)) + SUM(LENGTH(timeline)), 0)` on `pages WHERE chunker_version < MARKDOWN_CHUNKER_VERSION`. `formatReembedPrompt(est, graceSeconds)` is the stderr-line formatter. `runPostUpgradeReembedPrompt(engine, model, opts)` orchestrates the 10-second Ctrl-C window; TTY-only wait (non-TTY auto-proceeds); `GBRAIN_NO_REEMBED=1` bails with a doctor-warning marker; `GBRAIN_REEMBED_GRACE_SECONDS=0` skips the wait.
- `src/commands/reindex.ts` ŌĆö `gbrain reindex --markdown [--limit N] [--dry-run] [--json] [--no-embed] [--repo PATH]`. Walks `pages WHERE page_kind = 'markdown' AND chunker_version < MARKDOWN_CHUNKER_VERSION` in 100-row batches, ordered by id. Rows with non-null `source_path` re-import via `importFromFile`; rows without fall back to `importFromContent` against stored `compiled_truth`. Both paths pass `forceRechunk: true` to bypass the `content_hash` short-circuit. Idempotent ŌĆö partial-completion re-runs resume via id-ordered batches. Wired into `src/commands/upgrade.ts:runPostUpgrade` after `apply-migrations`.
- `src/commands/reindex-code.ts` ŌĆö `gbrain reindex --code [--source ID] [--dry-run] [--yes] [--json] [--force] [--no-embed]`. Walks `pages WHERE type = 'code'` in 100-row batches, replays through `importCodeFile` for chunk + embed + content_hash folding. Idempotent unless `--force` bypasses the content_hash early-return. Cost-preview model field reads `getEmbeddingModelName()` from the gateway. `runReindexCode` prints an informational stderr nudge when the configured embedding model isn't code-tuned (allowlist `{'voyage-code-3'}`, case-insensitive bare match), recommending `voyage:voyage-code-3`. Suppress with `GBRAIN_NO_CODE_MODEL_NUDGE=1`, `--no-embed`, or `--json`. Pure `shouldNudgeCodeModel(bareName)` helper returns a tagged `NudgeDecision` union.
- `src/commands/sync.ts:resolveSlugByPathOrSourcePath` ŌĆö Resolves a slug by `pages.source_path` first (returns the stored slug for frontmatter-fallback pages whose path doesn't derive a slug), then falls back to `resolveSlugForPath(path)`. Threaded into all 4 delete/rename call sites so emoji-only / Thai / Arabic filenames whose slug came from frontmatter don't orphan on delete/rename. Best-effort query ŌĆö pre-migration brains fall through to the legacy path.
- `src/core/utils.ts` ŌĆö Shared SQL utilities extracted from postgres-engine.ts. Exports `parseEmbedding(value)` (throws on unknown input; used by migration + ingest paths), `tryParseEmbedding(value)` (returns `null` + warns once per process; used by search/rescore paths), `isUndefinedColumnError(err)` (matches Postgres SQLSTATE 42703 / "column ... does not exist" with driver shape variation tolerated, so genuine errors propagate while column-missing falls through to legacy fallback), and `validateSourceId(id)` (throws on anything outside `^[a-z0-9_-]+$`; guards per-source disk-layout joins against path traversal). `rowToPage` populates the required `Page.source_id` field from the SELECT projection (`scripts/check-source-id-projection.sh` enforces every feeding projection includes the column).
- `src/core/db.ts` ŌĆö Connection management, schema initialization. `resolveSessionTimeouts()` returns `statement_timeout` + `idle_in_transaction_session_timeout` (defaults 5min each, env-overridable via `GBRAIN_STATEMENT_TIMEOUT` / `GBRAIN_IDLE_TX_TIMEOUT` / `GBRAIN_CLIENT_CHECK_INTERVAL`). Both `connect()` (module singleton) and `PostgresEngine.connect()` (worker pool) consume it via postgres.js's `connection` option, sending GUCs as startup parameters that survive PgBouncer transaction mode.
- `src/commands/migrate-engine.ts` ŌĆö Bidirectional engine migration (`gbrain migrate --to supabase/pglite`).
- `src/core/import-file.ts` ŌĆö importFromFile + importFromContent (chunk + embed + tags).
- `src/core/sync.ts` ŌĆö Pure sync functions (manifest parsing, filtering, slug conversion). Exports `pruneDir(name: string): boolean` ŌĆö single source of truth for descent-time directory exclusion across walkers (blocks `node_modules`, dot-prefix dirs, `ops/`, `*.raw` sidecars). `isSyncable` applies it per path segment; `walkMarkdownFiles` (extract.ts) and `listTextFiles` (transcript-discovery.ts) consult it before recursing. `manageGitignore` worktree discriminator matches the gitdir path segment (`/modules/<name>` = submodule, `/worktrees/<name>` = worktree). `classifyErrorCode(errorMsg)` regex classifier with 12 codes (`SLUG_MISMATCH`, `YAML_PARSE`, `YAML_DUPLICATE_KEY`, `MISSING_OPEN`, `MISSING_CLOSE`, `NESTED_QUOTES`, `EMPTY_FRONTMATTER`, `NULL_BYTES`, `INVALID_UTF8`, `STATEMENT_TIMEOUT`, `FILE_TOO_LARGE`, `SYMLINK_NOT_ALLOWED`) plus `UNKNOWN`. `summarizeFailuresByCode(failures)` returns sorted `[{code, count}]`. `SyncFailure` has optional `code?`; `acknowledgeSyncFailures()` returns `AcknowledgeResult { count, summary }`.
- `src/core/storage.ts` ŌĆö Pluggable storage interface (S3, Supabase Storage, local).
- `src/core/storage-config.ts` ŌĆö Storage tiering: `loadStorageConfig` reads `gbrain.yml`, normalizes deprecated keys (`git_tracked` / `supabase_only`) to canonical (`db_tracked` / `db_only`) with once-per-process deprecation warning, and runs `normalizeAndValidateStorageConfig` (auto-fixes missing trailing `/`, throws `StorageConfigError` on tier overlap). Path-segment matcher: `media/x/` does NOT match `media/xerox/foo`. Dedicated parser for the `gbrain.yml` shape (not gray-matter).
- `src/core/disk-walk.ts` ŌĆö `walkBrainRepo(repoPath)` returns `Map<slug, {size, mtimeMs}>` from one recursive `readdirSync`. Skips dot-dirs, `node_modules`, non-`.md` files. Used by `gbrain storage status`.
- `src/core/git-remote.ts` ŌĆö SSRF-hardened git invocations for remote-source `cloneRepo` and `pullRepo`. Exports two distinct flag constants: `GIT_SSRF_FLAGS` (3 `-c` config flags ŌĆö `protocol.allow=user`, `protocol.file.allow=never`, `http.allowRedirects=false`) is **global config**, spread BEFORE the subcommand verb; `GIT_SSRF_SUBCOMMAND_FLAGS = ['--no-recurse-submodules']` is **subcommand-scoped**, spread AFTER the verb. `cloneRepo` argv: `git <GIT_SSRF_FLAGS> clone <GIT_SSRF_SUBCOMMAND_FLAGS> --depth=1 [--branch X] -- <url> <dir>`. `pullRepo` argv: `git <GIT_SSRF_FLAGS> -C <dir> pull <GIT_SSRF_SUBCOMMAND_FLAGS> --ff-only`.
- `src/commands/storage.ts` ŌĆö `gbrain storage status [--repo P] [--json]`. Split into pure data (`getStorageStatus`) + JSON formatter + human formatter (ASCII-only). `PageCountsByTier` and `DiskUsageByTier` are distinct nominal types so swaps fail at compile time.
- `gbrain.yml` (brain repo root) ŌĆö Optional storage tiering config. Top-level `storage:` section with `db_tracked:` and `db_only:` array-valued keys. `gbrain sync` auto-manages `.gitignore` for `db_only` paths on successful sync (skips on dry-run, blocked-by-failures, submodule context, or `GBRAIN_NO_GITIGNORE=1`). `gbrain export --restore-only [--repo P] [--type T] [--slug-prefix S]` repopulates missing `db_only` files from the database.
- `src/core/supabase-admin.ts` ŌĆö Supabase admin API (project discovery, pgvector check).
- `src/core/file-resolver.ts` ŌĆö File resolution with fallback chain (local -> .redirect.yaml -> .redirect -> .supabase).
- `src/core/chunkers/` ŌĆö 3-tier chunking (recursive, semantic, LLM-guided). `code.ts` is a tree-sitter-based semantic chunker for 30+ languages (including SQL) with embedded-asset WASMs (`src/assets/wasm/`), `@dqbd/tiktoken` cl100k_base tokenizer, small-sibling merging. `CHUNKER_VERSION` constant folded into `importCodeFile`'s `content_hash` so chunker shape changes force clean re-chunks. `extractSymbolName` has an inline SQL branch (`extractSqlSymbolName`) diving through the `statement` wrapper into the inner DDL child (`create_table`/`create_function`/`create_view`/`create_index`/`create_procedure`/`create_type`/`create_schema`/`create_database`/`create_trigger`/`alter_table`/`alter_view`) and extracting the target identifier via the `name` field with identifier-shaped fallback; DML kinds return null. `normalizeSymbolType` has parallel SQL branches (`create_table ŌåÆ 'table'`, etc.).
- `src/core/errors.ts` ŌĆö `StructuredAgentError` + `buildError` + `serializeError`. Agent-facing surfaces (code-def, code-refs, usage errors) use this envelope; matches `CycleReport.PhaseResult.error` shape.
- `src/assets/wasm/` ŌĆö 37 tree-sitter grammar WASMs + tree-sitter runtime. Committed so `bun --compile` embeds them deterministically via `import path from ... with { type: 'file' }`. CI guard `scripts/check-wasm-embedded.sh` fails the build if the compiled binary silently falls through to recursive chunks. Includes `tree-sitter-sql.wasm` (DerekStride/tree-sitter-sql, covers PostgreSQL + MySQL + SQLite + T-SQL basics; ~11 MB).
- `src/commands/code-def.ts` + `src/commands/code-refs.ts` ŌĆö symbol definition + references lookup. Query `content_chunks.symbol_name` or chunk_text ILIKE with `page_kind='code'` filter. Auto-JSON when stdout is not a TTY. Bypass the standard `searchKeyword` `DISTINCT ON (slug)` collapse so multiple call-sites from the same file surface. `code-def.ts:DEF_TYPES` includes `'table' | 'view' | 'index' | 'procedure' | 'schema' | 'database' | 'trigger'`.
- `src/core/search/` ŌĆö Hybrid search: vector + keyword + RRF + multi-query expansion + dedup. `searchKeyword` / `searchKeywordChunks` / `searchVector` apply source-aware ranking at the SQL layer (curated content like `originals/`, `concepts/`, `writing/` outranks bulk content like `daily/`, `media/x/`). `searchVector` uses a two-stage CTE so source-boost re-ranking doesn't kill the HNSW index. Hard-exclude prefixes (`test/`, `archive/`, `attachments/`, `.raw/` by default) filter at retrieval, not post-rank. All gates honor `detail !== 'high'` so temporal queries surface chat pages normally.
- `src/core/search/intent.ts` ŌĆö Query intent classifier (entity/temporal/event/general ŌåÆ auto-selects detail level).
- `src/core/search/eval.ts` ŌĆö Retrieval eval harness: P@k, R@k, MRR, nDCG@k metrics + `runEval()` orchestrator.
- `src/core/search/source-boost.ts` ŌĆö Source-type boost map keyed by slug prefix. `DEFAULT_SOURCE_BOOSTS` (originals/ 1.5, concepts/ 1.3, writing/ 1.4, people/companies/deals/ 1.2, daily/ 0.8, media/x/ 0.7) and `DEFAULT_HARD_EXCLUDES` (test/, archive/, attachments/, .raw/). `parseSourceBoostEnv` / `parseHardExcludesEnv` parse comma-separated `prefix:factor` pairs from `GBRAIN_SOURCE_BOOST` / `GBRAIN_SEARCH_EXCLUDE`. `resolveBoostMap` / `resolveHardExcludes` merge defaults + env + caller `SearchOpts.exclude_slug_prefixes`/`include_slug_prefixes`.
- `src/core/search/sql-ranking.ts` ŌĆö Pure SQL string builders. `buildSourceFactorCase(slugColumn, boostMap, detail)` emits a CASE expression with longest-prefix-match wins (returns literal `'1.0'` when `detail === 'high'` for temporal-bypass parity). `buildHardExcludeClause(slugColumn, prefixes)` emits `NOT (col LIKE 'p1%' OR col LIKE 'p2%')`. LIKE meta-character escape covers `%`, `_`, AND `\`. Single-quote doubling on SQL string literals so injection-style inputs are inert text.

## Thin-client routing (v0.31.1, Issue #734)

`gbrain init --mcp-only` (v0.29.2) sets up a thin-client install: no local
brain content, just an OAuth client pointing at a remote `gbrain serve --http`.
v0.29.2/v0.30.0 only refused 9 obvious local-only commands; the other ~25
silently fell through to `connectEngine()` and opened the empty local PGLite,
returning "No results." against a populated remote brain. v0.31.1 fixes the
silent-empty-results bug class for every operation surface.
